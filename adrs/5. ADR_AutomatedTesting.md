# Automated Testing with GitHub Actions

## Context and Problem Statement

The project had a functional API and tests implemented using unittest, executed manually in the local environment.

There was no automatic mechanism to validate the correct functioning of the code upon changes, which implied the risk of introducing errors without early detection.

A solution was needed to run tests automatically on each repository modification, aligned with modern Continuous Integration (CI) practices.

## Decision Drivers

* Need to automatically validate code changes.
* Need to detect errors early in the development process.
* Need to align the project with Continuous Integration practices.
* Need to improve code quality and reliability.
* Need to establish a foundation for future automation improvements.

## Considered Options

* Continue executing tests manually.
* Implement a CI pipeline using GitHub Actions.
* Use Poetry for dependency management.

## Decision Outcome

Chosen option: "Implement a CI pipeline using GitHub Actions", because it provides automatic validation of the codebase on every repository change while integrating naturally with the existing GitHub workflow.

The workflow was configured to run on every push and pull request.

The pipeline installs dependencies using `requirements.txt` and executes the test suite with:

```bash
python -m unittest discover -s tests
```

The workflow is defined in:

```text
.github/workflows/tests.yml
```

### Consequences

* Good, because code is automatically validated on every change.
* Good, because errors can be detected earlier.
* Good, because it improves code quality and reliability.
* Good, because it provides a foundation for more advanced practices such as coverage analysis and linting.
* Good, because it aligns the project with professional development standards.

### Confirmation

The CI workflow automatically executes the test suite on every push and pull request, validating the correct behavior of the application before changes are integrated.

## Pros and Cons of the Options

### Continue executing tests manually

* Good, because it requires no additional infrastructure.
* Bad, because validation depends entirely on manual execution.
* Bad, because errors may go undetected until later stages.

### Implement a CI pipeline using GitHub Actions

* Good, because tests run automatically on repository changes.
* Good, because errors are detected earlier.
* Good, because it improves reliability and development practices.
* Good, because it creates a foundation for future CI enhancements.

### Use Poetry for dependency management

* Good, because it provides better dependency version control.
* Bad, because it adds additional complexity.
* Neutral, because the project already used an existing dependency management approach.

## More Information

The workflow executes the existing `unittest` test suite and integrates directly with the GitHub repository through GitHub Actions.
