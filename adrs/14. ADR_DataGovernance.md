# Data Governance with DataHub

## Context and Problem Statement

As the project evolved from a simple API into a complete data platform following a Medallion Architecture (Bronze, Silver, and Gold), managing datasets became increasingly complex.

Multiple data transformations, schemas, and dependencies were introduced through PostgreSQL and dbt. Without proper governance mechanisms, it would become difficult to understand data lineage, track ownership, maintain documentation, and ensure trust in the data assets.

The team required a centralized solution to provide metadata management, documentation, lineage visualization, and dataset discovery.

## Decision Drivers

- Improve data discoverability.
- Document datasets and their purpose.
- Visualize lineage across layers.
- Track dataset ownership.
- Increase trust in data products.
- Support future scalability of the data platform.

## Considered Options

### Maintain documentation manually

Document datasets using only markdown files and external documentation.

### Use dbt documentation only

Rely exclusively on dbt-generated documentation and lineage.

### Implement a dedicated data catalog

Adopt a metadata platform capable of centralizing documentation and lineage across the data stack.

## Decision Outcome

The team decided to implement DataHub as the central data governance platform.

DataHub was selected because it provides metadata management, lineage visualization, ownership assignment, dataset discovery, and seamless integration with PostgreSQL and dbt.

Metadata ingestion pipelines were configured to collect information from both the database and dbt artifacts.

## Metadata Ingestion

Two metadata sources were integrated:

### PostgreSQL Metadata

Metadata was ingested directly from PostgreSQL schemas, allowing DataHub to discover datasets and their structural information.

The following schemas were cataloged:

- `bronze`
- `silver`
- `gold`

### dbt Metadata

DataHub ingested dbt artifacts to enrich datasets with:

- Model descriptions
- Column descriptions
- Tests
- Data lineage
- Transformation metadata

This integration enabled the visualization of relationships between raw data sources and analytical models.

## Dataset Documentation

Documentation was added to datasets using dbt YAML files.

Descriptions were created for:

- Silver models
- Gold models
- Model columns

This documentation allows users to understand the purpose, business meaning, and usage of each dataset.

Source documentation was also created for Bronze datasets through `sources.yml`.

## Data Lineage

DataHub automatically generated lineage graphs showing the flow of data across the Medallion Architecture.

The lineage captures transformations from:

- Bronze datasets
- Silver cleaned datasets
- Gold dimensional models and facts

This allows users to understand how analytical datasets are derived from raw data sources.

## Ownership Management

Dataset owners were assigned to improve accountability and facilitate collaboration.

Ownership information allows users to identify responsible team members for each dataset and contact them when necessary.

## Data Quality Integration

DataHub integrates metadata generated by dbt tests, enabling visibility into data quality checks.

Examples of implemented tests include:

- `not_null`
- `unique`
- `relationships`
- `accepted_values`
- Custom business rules

This integration improves trust in the data by exposing validation results alongside dataset metadata.

## Consequences

### Positive Consequences

- Centralized metadata management.
- Improved dataset discoverability.
- Better documentation and collaboration.
- End-to-end lineage visualization.
- Increased trust in data products.
- Better support for future scaling.

### Negative Consequences

- Additional infrastructure to maintain.
- Metadata ingestion pipelines require synchronization.
- Documentation must be continuously updated.

## Confirmation

DataHub successfully ingests metadata from PostgreSQL and dbt, providing dataset discovery, documentation, ownership information, and lineage visualization across Bronze, Silver, and Gold layers.

The platform enables users to navigate the data ecosystem and better understand data dependencies and transformations.

## Pros and Cons of the Options

### Maintain documentation manually

**Pros**

- Simple implementation.
- No additional infrastructure.

**Cons**

- Difficult to maintain.
- Documentation becomes outdated easily.
- No lineage visualization.

### Use dbt documentation only

**Pros**

- Native integration with transformations.
- Automatic lineage between models.

**Cons**

- Limited metadata capabilities.
- No centralized governance platform.
- Reduced discoverability.

### Implement a dedicated data catalog

**Pros**

- Centralized metadata management.
- Ownership and lineage support.
- Better data discoverability.
- Integration with multiple systems.

**Cons**

- Additional operational complexity.
- Requires metadata ingestion configuration.

## More Information

DataHub was deployed using Docker and integrated with PostgreSQL and dbt metadata ingestion pipelines. The platform provides a centralized view of the entire Medallion Architecture and serves as the foundation for data governance within the project.
