# Data Ingestion into the Bronze Layer

## Context and Problem Statement

The project implements a Medallion Architecture for oil and gas production data. The Bronze layer is responsible for preserving the raw source datasets exactly as they are received, so that later transformations in the Silver and Gold layers can be reproduced.

The initial data load had to ingest official datasets from Argentina's public energy data portal (datos.energia.gob.ar) into the project repository and then make them available in PostgreSQL under the bronze schema.

Without a reproducible ingestion process, the project would depend on manual downloads, local files with unclear origin, and non repeatable steps. This would reduce traceability and make future orchestration difficult.

## Decision Drivers

- Preserve official source data in its raw format.
- Ensure the ingestion process is reproducible and implemented as code.
- Avoid manual downloads as part of the regular pipeline.
- Keep the Bronze layer independent from cleansing and transformation logic.
- Support future automated orchestration workflows.
- Maintain traceability from the public source datasets to analytical outputs.

## Considered Options

### Option 1: Manual download of CSV files

Download the official CSV files manually from the public data portal and place them in the Bronze directory.

**Pros**
- Simple for the initial exploration phase.
- No additional implementation required.

**Cons**
- Not reproducible.
- Depends on manual execution.
- Source URLs and download logic remain undocumented in code.
- Difficult to automate in a pipeline.
- Higher risk of stale or inconsistent local files.

### Option 2: Reproducible extraction script

Implement a script that downloads the official datasets from datos.energia.gob.ar and stores them in the Bronze layer.

**Pros**
- Reproducible and versionable ingestion logic.
- Source URLs are explicitly documented in code.
- Easy to execute locally and from orchestration tools.
- Preserves the raw CSV files before transformation.
- Aligns with the Medallion Architecture.

**Cons**
- Requires network availability when executed.
- Requires maintenance if source URLs change.
- Does not provide advanced orchestration by itself.

## Decision Outcome

We adopted a reproducible extraction process implemented in code as the primary mechanism for data ingestion. The ingestion process downloads the official datasets from datos.energia.gob.ar into the Bronze layer as raw CSV files:
- wells_production.csv
- listed_wells.csv

The files are stored under: data/bronze/

After extraction, the Bronze loading process reads the raw CSV files and loads them into PostgreSQL under the `bronze` schema:

- `bronze.wells_production`
- `bronze.listed_wells`

The Bronze layer does not apply cleansing, deduplication, type standardization, or business transformations. Those responsibilities belong to the Silver and Gold layers.

## Implementation Details

The extraction step is implemented in:
src/ingestion/extract_data.py

This script downloads the official public datasets and writes them to data/bronze.

## Decision Outcome

The selected approach provides a reproducible and auditable ingestion process that preserves official source datasets in the Bronze layer before any transformation is applied.

It also creates a clear separation between extraction, raw storage, database loading, transformation, and validation. This makes the pipeline easier to maintain and prepares it for automated orchestration.

### Consequences

#### Positive

- Raw source data is preserved for auditing and reprocessing.
- Ingestion logic is explicit, reproducible, and version controlled.
- Manual downloads are avoided during regular pipeline execution.
- The Bronze layer remains isolated from transformation logic.
- PostgreSQL Bronze tables can be consumed directly by dbt sources.
- The process can be executed locally or orchestrated with Airflow.

#### Negative

- The ingestion process depends on availability of the external public data portal.
- Source URL changes require updates to the extraction script.
- Replacing Bronze tables on each load may remove previous raw snapshots unless versioning is added later.
- Additional orchestration and monitoring are required for production grade reliability.
