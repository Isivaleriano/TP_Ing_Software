# Deployment on AWS EC2

## Context and Problem Statement

Once the API was containerized with Docker, a publicly accessible hosting environment was needed for the teaching team to evaluate the service during the correction period.

A simple, cost-effective solution compatible with Docker was required.

## Decision Drivers

* Need to provide a publicly accessible environment for evaluation.
* Need a solution compatible with Docker.
* Need to keep infrastructure costs low.
* Need a deployment environment that can be configured quickly.
* Need a foundation for future scalability.

## Considered Options

* Deploy the application on an AWS EC2 instance.
* Use AWS ECS for container execution.

## Decision Outcome

Chosen option: "Deploy the application on an AWS EC2 instance", because it provides a simple and cost-effective solution compatible with Docker while remaining within the free tier.

The API was deployed on an AWS EC2 instance (`t2.micro`, Ubuntu 24.04 LTS).

The Docker image is pulled from GitHub Container Registry (`ghcr.io`) and executed exposing port `8000`.

Ports `22` (SSH) and `8000` (API) were enabled in the Security Group.

### Consequences

* Good, because it provides a cost-effective solution within the AWS free tier.
* Good, because it offers full control over the infrastructure.
* Good, because it is compatible with Docker without requiring additional configuration.
* Good, because it provides a foundation for future scaling.

### Confirmation

The API was successfully deployed on an AWS EC2 instance and made publicly accessible for evaluation purposes.

## Pros and Cons of the Options

### Deploy the application on an AWS EC2 instance

* Good, because it is compatible with Docker.
* Good, because it provides full infrastructure control.
* Good, because it can be used within the AWS free tier.
* Good, because it is simple to configure for the project scope.

### Use AWS ECS for container execution

* Good, because it is specifically designed for running containers.
* Neutral, because it could be considered for future phases.
* Bad, because it introduces greater configuration complexity for the scope of the project.

## More Information

The deployment uses an EC2 instance running Ubuntu 24.04 LTS and Docker, with the application image hosted in GitHub Container Registry (`ghcr.io`).
